## Протоколы

Обязательные/Необязательные



Именование протоклов:

- Оканчиваются на **~able**: `JSONSerialalizable`, `CustomStringConvertible`. Определяют интерфейс для функциональности
- Оканчивается на **~Delegale**: `KitchenViewControllerDelegate`. Играют роль посредников.

CustomStringConvertible



### **~able**

`CustomStringConvertible` - встроенный протокол, позволяет определить текстовое представление объекта через свойство `description: String`





### ~Delegale

Как правило, протокол определяется в том же файле, где реализован тип который под этот протокол должен подписываться.

Название протокола определяется названием того класса в котором будет инициализирован экземпляр этого протокола и в конце добавляется Delegale.

```swift
// файл KitchenViewController.swift

final class KitchenViewController: UIViewController {
  weak var delegate: KitchenViewControllerDelegate?
  func compliteOrderButtonPressed() {
    delegate?.completeOrder()
  }
}
```

```swift
// файл ClientViewController.swift

protocol KitchenViewControllerDelegate: AnyObject {
  func completeOrder()
}

final class ClientViewController: UIViewController {
  override func prepare(for segue: ...) {
    guard let kitchenVC = segue.destination as? KitchenViewController else { return }
    kitchenVC.delegate = self
  }
}

extension ClientViewController: KitchenViewControllerDelegate {
  func completeOrder() {
    // Обработчик
  }
}
```
